CI-Specific Karma Configuration

karma-ci.conf.js
const karmaConfig = require('./karma.conf.js');

module.exports = function (config) {
  karmaConfig(config);
  
  config.set({
    // Use headless Chrome for CI
    browsers: ['ChromeHeadlessCI'],
    customLaunchers: {
      ChromeHeadlessCI: {
        base: 'ChromeHeadless',
        flags: [
          '--no-sandbox',
          '--disable-web-security',
          '--disable-gpu',
          '--remote-debugging-port=9222'
        ]
      }
    },
    
    // Single run for CI
    singleRun: true,
    autoWatch: false,
    
    // Reporters for CI
    reporters: ['progress', 'junit', 'coverage'],
    
    // JUnit reporter for Azure DevOps
    junitReporter: {
      outputDir: './test-results/',
      outputFile: 'test-results.xml',
      useBrowserName: false
    },
    
    // Coverage reporter
    coverageReporter: {
      dir: './coverage/',
      reporters: [
        { type: 'html' },
        { type: 'lcov' },
        { type: 'json-summary' }
      ]
    }
  });
};

🚀 Caching Strategy

Our pipeline implements a multi-level caching strategy to optimize build times and resource usage:

Cache Layers

1. Node Modules Cache

Purpose: Cache npm packages to avoid repeated downloads

Key Strategy: Based on package-lock.json hash + OS + branch

Expected Speed Improvement: 60-80% faster dependency installation

2. Angular Build Cache

Purpose: Cache Angular compiler output

Location: .angular/cache directory

Expected Speed Improvement: 40-60% faster builds

3. NPM Global Cache

Purpose: Cache global npm packages like Angular CLI

Key Strategy: Based on Node.js version

Expected Speed Improvement: 20-30% faster CLI installation

Cache Configuration Examples

Node Modules Cache Configuration
- task: Cache@2
  displayName: 'Cache node_modules'
  inputs:
    key: 'npm-$(Agent.OS)-$(Build.SourceBranch)-"$(Build.SourceVersion)"-$(Agent.JobName)'
    restoreKeys: |
      npm-$(Agent.OS)-$(Build.SourceBranch)-"$(Build.SourceVersion)"
      npm-$(Agent.OS)-$(Build.SourceBranch)
      npm-$(Agent.OS)
    path: 'node_modules'
    cacheHitVar: 'NODE_MODULES_RESTORED'
Angular Build Cache Configuration
- task: Cache@2
  displayName: 'Cache Angular Build Cache'
  inputs:
    key: 'angular-cache-$(Agent.OS)-$(Build.SourceBranch)-"$(Build.SourceVersion)"'
    restoreKeys: |
      angular-cache-$(Agent.OS)-$(Build.SourceBranch)
      angular-cache-$(Agent.OS)
    path: '.angular/cache'

Cache Performance Metrics
