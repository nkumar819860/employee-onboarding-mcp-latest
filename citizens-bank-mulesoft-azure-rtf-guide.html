appName

Format: 'citizens-banking-integration' | Pattern: [a-z-]+
appVersion
Application version with build number
Format: '1.0.0-$(Build.BuildNumber)' | SemVer compliant
projectGroupId
Maven group ID for organization
Format: 'com.citizensbank.integration' | Reverse domain notation

Azure RTF Configuration

rtfClusterName
Azure Kubernetes Service cluster name for RTF
Example: 'citizens-bank-rtf-prod' | Must exist in Azure
rtfNamespace
Kubernetes namespace for MuleSoft applications
Default: 'citizens-banking-apps' | Lowercase, hyphenated
rtfReplicas
Number of application replicas for high availability
Default: '3' | Minimum: 2, Recommended: 3-5 for banking
rtfCpuCores
CPU cores allocation per replica
Default: '2' | Options: '1', '2', '4', '8'
rtfMemoryLimit
Memory limit per replica
Default: '2Gi' | Options: '1Gi', '2Gi', '4Gi', '8Gi'

📋 Prerequisites & Setup Requirements

☁️ Azure Infrastructure Requirements

1

Azure Kubernetes Service (AKS) Cluster

A properly configured AKS cluster with Runtime Fabric installed for Citizens Bank applications:

  • ✅ Kubernetes version 1.24 or higher
  • ✅ Minimum 5 nodes with 8 vCPUs and 32GB RAM each (Banking requirements)
  • ✅ Premium storage class configured with encryption
  • ✅ Network connectivity to Anypoint Platform
  • ✅ Container registry access configured with security scanning
# Create AKS cluster for Citizens Bank with enhanced security az aks create \ --resource-group citizens-bank-mulesoft-rg \ --name citizens-bank-rtf-prod \ --node-count 5 \ --node-vm-size Standard_D8s_v3 \ --kubernetes-version 1.28 \ --enable-addons monitoring,azure-policy \ --network-policy azure \ --enable-encryption-at-host \ --generate-ssh-keys
2

Citizens Bank Azure DevOps Configuration

Configure Azure DevOps with proper service connections for banking compliance:

  • ✅ Azure Resource Manager service connection with restricted permissions
  • ✅ Kubernetes service connection to AKS cluster
  • ✅ Secure Docker registry connection with vulnerability scanning
  • ✅ RBAC permissions configured for least privilege access
  • ✅ Variable groups created with appropriate encryption
⚠️ Banking Compliance RBAC Configuration: Ensure the service principal has:
  • Azure Kubernetes Service Cluster Admin Role (limited scope)

  • Security Reader for compliance monitoring

Runtime Fabric Installation & Banking Security Configuration
