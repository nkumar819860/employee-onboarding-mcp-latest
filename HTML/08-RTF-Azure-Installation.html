🌐 Runtime Fabric Installation on Microsoft Azure

Complete step-by-step guide for deploying MuleSoft Runtime Fabric on Azure Kubernetes Service (AKS)

🏠 Home ☁️ AWS Setup 🔐 Security 📊 Monitoring

📋 Prerequisites & Requirements

🔑 Azure Requirements

  • Azure Subscription with sufficient credits
  • Contributor or Owner role permissions
  • Resource quota for AKS cluster
  • Azure CLI 2.30+ installed

🛠️ Tools & Software

  • kubectl 1.24+ command-line tool
  • Helm 3.8+ for package management
  • Docker Desktop (for local testing)
  • Git for version control

🎯 MuleSoft Requirements

  • Anypoint Platform organization access
  • Runtime Fabric license from MuleSoft
  • Environment administrator permissions
  • Connected App for API access

🏗️ Azure Infrastructure Setup

🌐 Step 1: Resource Group Creation

# Set variables for consistent naming export RESOURCE_GROUP="rtf-production-rg" export LOCATION="eastus2" export CLUSTER_NAME="rtf-aks-cluster" # Create resource group az group create \ --name $RESOURCE_GROUP \ --location $LOCATION \ --tags environment=production project=runtime-fabric # Verify creation az group show --name $RESOURCE_GROUP --output table

🔗 Step 2: Virtual Network Configuration

# Create virtual network with proper CIDR blocks az network vnet create \ --resource-group $RESOURCE_GROUP \ --name rtf-vnet \ --address-prefix 10.0.0.0/16 \ --subnet-name rtf-subnet \ --subnet-prefix 10.0.1.0/24 \ --location $LOCATION # Create additional subnets for different services az network vnet subnet create \ --resource-group $RESOURCE_GROUP \ --vnet-name rtf-vnet \ --name services-subnet \ --address-prefix 10.0.2.0/24 az network vnet subnet create \ --resource-group $RESOURCE_GROUP \ --vnet-name rtf-vnet \ --name monitoring-subnet \ --address-prefix 10.0.3.0/24 # Get subnet ID for AKS creation SUBNET_ID=$(az network vnet subnet show \ --resource-group $RESOURCE_GROUP \ --vnet-name rtf-vnet \ --name rtf-subnet \ --query id -o tsv) echo "Subnet ID: $SUBNET_ID"

🛡️ Step 3: Network Security Configuration

# Create Network Security Group az network nsg create \ --resource-group $RESOURCE_GROUP \ --name rtf-nsg \ --location $LOCATION # Add security rules for RTF az network nsg rule create \ --resource-group $RESOURCE_GROUP \ --nsg-name rtf-nsg \ --name AllowHTTPS \ --protocol tcp \ --priority 1001 \ --destination-port-range 443 \ --source-address-prefix '*' \ --destination-address-prefix '*' \ --access allow \ --direction inbound az network nsg rule create \ --resource-group $RESOURCE_GROUP \ --nsg-name rtf-nsg \ --name AllowRTFManagement \ --protocol tcp \ --priority 1002 \ --destination-port-range 32009 \ --source-address-prefix VirtualNetwork \ --destination-address-prefix '*' \ --access allow \ --direction inbound # Associate NSG with subnet az network vnet subnet update \ --resource-group $RESOURCE_GROUP \ --vnet-name rtf-vnet \ --name rtf-subnet \ --network-security-group rtf-nsg

☸️ Azure Kubernetes Service (AKS) Setup

🚀 Step 1: Create AKS Cluster

💡 Recommended Configuration: This setup creates a production-ready AKS cluster with auto-scaling, monitoring, and proper networking configuration.
# Create AKS cluster with production settings az aks create \ --resource-group $RESOURCE_GROUP \ --name $CLUSTER_NAME \ --node-count 3 \ --min-count 3 \ --max-count 10 \ --node-vm-size Standard_D4s_v3 \ --kubernetes-version 1.28.5 \ --enable-addons monitoring \ --vnet-subnet-id $SUBNET_ID \ --enable-cluster-autoscaler \ --enable-node-public-ip false \ --network-plugin azure \ --network-policy azure \ --generate-ssh-keys \ --enable-managed-identity \ --assign-identity \ --tags environment=production # This will take 10-15 minutes to complete echo "AKS cluster creation in progress..."

⚙️ Step 2: Configure kubectl Access

# Get AKS credentials and merge with local kubeconfig az aks get-credentials \ --resource-group $RESOURCE_GROUP \ --name $CLUSTER_NAME \ --overwrite-existing # Verify connection and cluster info kubectl get nodes -o wide kubectl cluster-info kubectl get namespaces # Check node resource capacity kubectl describe nodes | grep -A 5 "Capacity:\|Allocatable:"

🔧 Runtime Fabric Installation

  1. 📥 Download RTF Installation Package
    Access the latest RTF installer from Anypoint Platform or use the direct download method below.
    # Create installation directory mkdir -p ~/rtf-installation && cd ~/rtf-installation # Download the latest RTF installer (replace with your org-specific URL) curl -L "https://anypoint.mulesoft.com/runtimefabric/api/download/installer/latest" \ -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \ -o rtf-install-package.tar.gz # Extract the installation package tar -xzf rtf-install-package.tar.gz cd rtf-install-package # Verify contents ls -la
  2. 📝 Create Runtime Fabric Configuration
    # Create RTF namespace kubectl create namespace rtf-system # Create RTF configuration file cat < rtf-config.yaml apiVersion: install.runtime-fabric.mulesoft.com/v1beta1 kind: RuntimeFabric metadata: name: azure-rtf-cluster namespace: rtf-system spec: # Anypoint Platform connection details anypoint: organizationId: "YOUR_ORG_ID" environment: "production" region: "us-east-1" # Cluster configuration cluster: name: "azure-rtf-production" region: "eastus2" provider: "azure" # Storage configuration storage: storageClass: "rtf-premium-storage" capacity: "100Gi" # Networking configuration ingress: class: "nginx" annotations: nginx.ingress.kubernetes.io/ssl-redirect: "true" nginx.ingress.kubernetes.io/force-ssl-redirect: "true" tls: enabled: true secretName: "rtf-tls-secret" # Resource allocation for RTF components resources: replicas: 3 requests: cpu: "1000m" memory: "2Gi" limits: cpu: "2000m" memory: "4Gi" # Auto-scaling configuration autoscaling: enabled: true minReplicas: 3 maxReplicas: 10 targetCPUUtilizationPercentage: 70 targetMemoryUtilizationPercentage: 80 # Security settings security: podSecurityPolicy: enabled: true networkPolicy: enabled: true rbac: enabled: true # Monitoring and observability monitoring: prometheus: enabled: true grafana: enabled: true logging: level: "INFO" EOF
  3. 🎯 Install Runtime Fabric Components
    # Install RTF using the installer script ./install-rtf.sh --config rtf-config.yaml --namespace rtf-system # Alternative: Manual Helm installation helm repo add mulesoft https://repository.mulesoft.org/nexus/content/repositories/public-helm/ helm repo update # Install RTF with custom values helm install rtf-azure mulesoft/runtime-fabric \ --namespace rtf-system \ --values rtf-config.yaml \ --wait \ --timeout 30m \ --set global.provider=azure \ --set global.region=eastus2 # Monitor installation progress kubectl get pods -n rtf-system -w
  4. ✅ Verify RTF Installation
    # Check all RTF pods are running kubectl get pods -n rtf-system -o wide # Verify RTF services kubectl get services -n rtf-system # Check RTF ingress configuration kubectl get ingress -n rtf-system # Verify RTF agent connection to Anypoint Platform kubectl logs -n rtf-system -l app=rtf-agent --tail=50 # Check cluster status from RTF perspective kubectl get rtf -n rtf-system # Verify storage claims kubectl get pvc -n rtf-system # Check node resources and allocations kubectl top nodes kubectl top pods -n rtf-system

🌐 Load Balancer and Ingress Configuration

🔧 Step 1: Install NGINX Ingress Controller

# Add NGINX ingress Helm repository helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx helm repo update # Create namespace for ingress kubectl create namespace ingress-nginx # Install NGINX ingress controller with Azure-specific settings helm install ingress-nginx ingress-nginx/ingress-nginx \ --namespace ingress-nginx \ --set controller.service.type=LoadBalancer \ --set controller.service.loadBalancerSourceRanges="{0.0.0.0/0}" \ --set controller.service.annotations."service\.beta\.kubernetes\.io/azure-load-balancer-health-probe-request-path"=/healthz \ --set controller.metrics.enabled=true \ --set controller.podSecurityContext.runAsUser=101 \ --set controller.image.allowPrivilegeEscalation=false # Wait for LoadBalancer to get external IP kubectl get service -n ingress-nginx ingress-nginx-controller --watch # Get the external IP address EXTERNAL_IP=$(kubectl get service -n ingress-nginx ingress-nginx-controller -o jsonpath='{.status.loadBalancer.ingress[0].ip}') echo "External IP: $EXTERNAL_IP"

📊 Monitoring and Observability Setup

📈 Azure Monitor Integration

# Enable Azure Monitor for containers if not already enabled az aks enable-addons \ --resource-group $RESOURCE_GROUP \ --name $CLUSTER_NAME \ --addons monitoring # Create dedicated Log Analytics workspace for RTF az monitor log-analytics workspace create \ --resource-group $RESOURCE_GROUP \ --workspace-name rtf-log-analytics \ --location $LOCATION \ --sku pergb2018 # Get workspace ID for configuration WORKSPACE_ID=$(az monitor log-analytics workspace show \ --resource-group $RESOURCE_GROUP \ --workspace-name rtf-log-analytics \ --query customerId -o tsv) echo "Log Analytics Workspace ID: $WORKSPACE_ID"

🎯 Prometheus and Grafana Setup

# Add Prometheus community Helm repository helm repo add prometheus-community https://prometheus-community.github.io/helm-charts helm repo add grafana https://grafana.github.io/helm-charts helm repo update # Create monitoring namespace kubectl create namespace monitoring # Install Prometheus stack with custom values helm install kube-prometheus prometheus-community/kube-prometheus-stack \ --namespace monitoring \ --set grafana.service.type=LoadBalancer \ --set grafana.persistence.enabled=true \ --set grafana.persistence.size=10Gi \ --set graf
