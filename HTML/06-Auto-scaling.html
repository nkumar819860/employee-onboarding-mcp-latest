📊 Horizontal Pod Autoscaler Configuration

🔧 Enhanced HPA with 80% Thresholds

# Advanced HPA Configuration for Employee Onboarding Platform
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: employee-onboarding-hpa
  namespace: employee-onboarding
  annotations:
    predictive.autoscaling/enabled: "true"
    predictive.autoscaling/model: "time-series"
    predictive.autoscaling/forecast-window: "30m"
    hipaa.compliance/data-classification: "pii-sensitive"
    hipaa.compliance/audit-enabled: "true"
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: employee-onboarding-mcp
  minReplicas: 3
  maxReplicas: 12
  
  # Resource Metrics with 80% Threshold
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 80
  - type: Resource
    resource:
      name: memory
      target:
        type: Utilization
        averageUtilization: 80
        
  # Custom Business Metrics
  - type: Pods
    pods:
      metric:
        name: employee_onboarding_requests_per_second
      target:
        type: AverageValue
        averageValue: "100"

  # Advanced Scaling Behavior
  behavior:
    scaleUp:
      stabilizationWindowSeconds: 60
      policies:
      - type: Percent
        value: 100
        periodSeconds: 15
      - type: Pods
        value: 3
        periodSeconds: 60
      selectPolicy: Max
    scaleDown:
      stabilizationWindowSeconds: 300
      policies:
      - type: Percent
        value: 20
        periodSeconds: 60
      - type: Pods
        value: 1
        periodSeconds: 300
      selectPolicy: Min

🔔 Monitoring & Alerts

📊 Prometheus Alerting Rules

# Prometheus AlertManager Rules
groups:
- name: employee-onboarding-autoscaling
  rules:
  - alert: HighCPUUtilization80Percent
    expr: avg(rate(container_cpu_usage_seconds_total[5m])) by (pod) * 100 > 80
    for: 2m
    labels:
      severity: warning
      component: employee-onboarding
      threshold: "80%"
    annotations:
      summary: "CPU usage exceeded 80% threshold"
      description: "Pod {{ $labels.pod }} CPU: {{ $value }}%"

💡 Auto-scaling Best Practices

⚠️ Important Considerations

  • Resource Limits: Always set appropriate resource limits to prevent resource starvation
  • Graceful Shutdown: Configure proper shutdown hooks for clean pod termination
  • Health Checks: Implement comprehensive readiness and liveness probes
  • Monitoring: Set up comprehensive monitoring and alerting
  • Testing: Regularly test scaling
