☁️ Runtime Fabric Installation on Amazon Web Services

Complete step-by-step guide for deploying MuleSoft Runtime Fabric on Amazon Elastic Kubernetes Service (EKS)

🏠 Home 🌐 Azure Setup 🔐 Security 📊 Monitoring

📋 Prerequisites & Requirements

☁️ AWS Requirements

  • AWS Account with appropriate permissions
  • IAM User with EKS, EC2, and VPC permissions
  • AWS CLI v2.0+ installed and configured
  • Service quota for EKS clusters and EC2 instances

🛠️ Tools & Software

  • kubectl 1.24+ command-line tool
  • eksctl for EKS cluster management
  • Helm 3.8+ for package management
  • Docker Desktop (for testing)

🎯 MuleSoft Requirements

  • Anypoint Platform organization access
  • Runtime Fabric license from MuleSoft
  • Environment administrator permissions
  • Connected App for secure API access

🔧 AWS CLI and Tools Setup

🚀 Step 1: Install and Configure AWS CLI

# Install AWS CLI v2 (Linux/macOS) curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip" unzip awscliv2.zip sudo ./aws/install # For Windows (PowerShell) # Download and install from: https://aws.amazon.com/cli/ # Configure AWS CLI with your credentials aws configure # Set the following when prompted: # AWS Access Key ID: [Your Access Key] # AWS Secret Access Key: [Your Secret Key] # Default region name: us-east-1 # Default output format: json # Verify configuration aws sts get-caller-identity

⚙️ Step 2: Install eksctl and kubectl

# Install eksctl (Linux/macOS) curl --silent --location "https://github.com/weaveworks/eksctl/releases/latest/download/eksctl_$(uname -s)_amd64.tar.gz" | tar xz -C /tmp sudo mv /tmp/eksctl /usr/local/bin # Install kubectl curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl" sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl # Verify installations eksctl version kubectl version --client # Install Helm curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash helm version
💡 AWS Regions: Choose a region that supports EKS and has the required instance types. Popular choices are us-east-1, us-west-2, or eu-west-1.

🏗️ AWS Infrastructure Setup

🌐 Step 1: Create VPC and Networking

# Set environment variables export AWS_REGION="us-east-1" export CLUSTER_NAME="rtf-eks-cluster" export VPC_CIDR="10.0.0.0/16" # Create VPC using CloudFormation template aws cloudformation create-stack \ --stack-name rtf-vpc-stack \ --template-url https://amazon-eks.s3.us-west-2.amazonaws.com/cloudformation/2020-10-29/amazon-eks-vpc-private-subnets.yaml \ --parameters ParameterKey=VpcBlock,ParameterValue=$VPC_CIDR \ --region $AWS_REGION # Wait for VPC stack to complete aws cloudformation wait stack-create-complete \ --stack-name rtf-vpc-stack \ --region $AWS_REGION # Get VPC and subnet IDs VPC_ID=$(aws cloudformation describe-stacks \ --stack-name rtf-vpc-stack \ --query 'Stacks[0].Outputs[?OutputKey==`VpcId`].OutputValue' \ --output text --region $AWS_REGION) echo "VPC ID: $VPC_ID"

🔐 Step 2: IAM Roles and Policies

# Create EKS cluster service role cat > eks-cluster-role-trust-policy.json << EOF { "Version": "2012-10-17", "Statement": [ { "Effect": "Allow", "Principal": { "Service": "eks.amazonaws.com" }, "Action": "sts:AssumeRole" } ] } EOF # Create the cluster service role aws iam create-role \ --role-name rtf-eks-cluster-role \ --assume-role-policy-document file://eks-cluster-role-trust-policy.json # Attach required policies aws iam attach-role-policy \ --role-name rtf-eks-cluster-role \ --policy-arn arn:aws:iam::aws:policy/AmazonEKSClusterPolicy # Create node group role cat > eks-nodegroup-role-trust-policy.json << EOF { "Version": "2012-10-17", "Statement": [ { "Effect": "Allow", "Principal": { "Service": "ec2.amazonaws.com" }, "Action": "sts:AssumeRole" } ] } EOF aws iam create-role \ --role-name rtf-eks-nodegroup-role \ --assume-role-policy-document file://eks-nodegroup-role-trust-policy.json # Attach policies to node group role aws iam attach-role-policy \ --role-name rtf-eks-nodegroup-role \ --policy-arn arn:aws:iam::aws:policy/AmazonEKSWorkerNodePolicy aws iam attach-role-policy \ --role-name rtf-eks-nodegroup-role \ --policy-arn arn:aws:iam::aws:policy/AmazonEKS_CNI_Policy aws iam attach-role-policy \ --role-name rtf-eks-nodegroup-role \ --policy-arn arn:aws:iam::aws:policy/AmazonEC2ContainerRegistryReadOnly

☸️ Amazon EKS Cluster Creation

🚀 Step 1: Create EKS Cluster using eksctl

💡 Production Configuration: This setup creates a production-ready EKS cluster with auto-scaling, managed node groups, and proper security configurations.
# Create cluster configuration file cat > rtf-eks-cluster.yaml << EOF apiVersion: eksctl.io/v1alpha5 kind: ClusterConfig metadata: name: $CLUSTER_NAME region: $AWS_REGION version: "1.28" # VPC Configuration vpc: cidr: "$VPC_CIDR" nat: gateway: Single clusterEndpoints: publicAccess: true privateAccess: true # IAM Configuration iam: serviceRoleARN: "arn:aws:iam::$(aws sts get-caller-identity --query Account --output text):role/rtf-eks-cluster-role" # Managed Node Groups managedNodeGroups: - name: rtf-workers instanceType: m5.xlarge amiFamily: AmazonLinux2 minSize: 3 maxSize: 10 desiredCapacity: 3 volumeSize: 100 volumeType: gp3 privateNetworking: true ssh: enableSsm: true iam: instanceRoleARN: "arn:aws:iam::$(aws sts get-caller-identity --query Account --output text):role/rtf-eks-nodegroup-role" tags: Environment: production Project: runtime-fabric labels: role: rtf-worker # Addons addons: - name: vpc-cni version: latest - name: coredns version: latest - name: kube-proxy version: latest - name: aws-ebs-csi-driver version: latest # CloudWatch Logging cloudWatch: clusterLogging: enable: ["all"] logRetentionInDays: 30 EOF # Create the cluster (this takes 15-20 minutes) eksctl create cluster -f rtf-eks-cluster.yaml echo "EKS cluster creation initiated. This will take 15-20 minutes..."

⚙️ Step 2: Verify Cluster and Configure Access

# Update kubeconfig aws eks update-kubeconfig \ --region $AWS_REGION \ --name $CLUSTER_NAME # Verify cluster access kubectl get nodes -o wide kubectl get pods --all-namespaces kubectl cluster-info # Check cluster status eksctl get cluster --region $AWS_REGION # Verify node group kubectl get nodes --show-labels kubectl describe nodes | grep -A 3 "Instance.*:"

💾 Step 3: Storage Classes Configuration

# Create high-performance storage class for RTF cat << EOF | kubectl apply -f - apiVersion: storage.k8s.io/v1 kind: StorageClass metadata: name: rtf-gp3-storage annotations: storageclass.kubernetes.io/is-default-class: "false" provisioner: ebs.csi.aws.com parameters: type: gp3 iops: "3000" throughput: "125" fsType: ext4 encrypted: "true" allowVolumeExpansion: true volumeBindingMode: WaitForFirstConsumer reclaimPolicy: Delete EOF # Create standard storage class cat << EOF | kubectl apply -f - apiVersion: storage.k8s.io/v1 kind: StorageClass metadata: name: rtf-gp2-storage provisioner: ebs.csi.aws.com parameters: type: gp2 fsType: ext4 encrypted: "true" allowVolumeExpansion: true volumeBindingMode: WaitForFirstConsumer reclaimPolicy: Delete EOF # Verify storage classes kubectl get storageclass

🔧 Runtime Fabric Installation

  1. 📥 Download RTF Installation Package
    Download the RTF installer from Anypoint Platform or use the MuleSoft CLI method.
    # Create installation directory mkdir -p ~/rtf-aws-installation && cd ~/rtf-aws-installation # Method 1: Download via Anypoint Platform UI # Navigate to Runtime Manager > Runtime Fabrics > Add Runtime Fabric # Download the installer package # Method 2: Use MuleSoft CLI (if available) # npm install -g @mulesoft/anypoint-cli # anypoint-cli runtime-fabric download --organizationId YOUR_ORG_ID # Extract the package (replace with actual filename) tar -xzf rtf-install-package-*.tar.gz cd rtf-install-package # Make scripts executable chmod +x *.sh ls -la
  2. 📝 Create Runtime Fabric Configuration
    # Create RTF namespace kubectl create namespace rtf-system # Create configuration file for AWS EKS cat << EOF > rtf-aws-config.yaml apiVersion: install.runtime-fabric.mulesoft.com/v1beta1 kind: RuntimeFabric metadata: name: aws-rtf-cluster namespace: rtf-system labels: environment: production provider: aws spec: # Anypoint Platform connection anypoint: organizationId: "YOUR_ORG_ID" environment: "production" region: "us-east-1" # AWS-specific cluster configuration cluster: name: "aws-rtf-production" region: "$AWS_REGION" provider: "aws" version: "1.28" # Storage configuration for AWS storage: storageClass: "rtf-gp3-storage" capacity: "100Gi" # Load balancer and ingress ingress: class: "alb" # AWS Application Load Balancer annotations: kubernetes.io/ingress.class: alb alb.ingress.kubernetes.io/scheme: internet-facing alb.ingress.kubernetes.io/target-type: ip alb.ingress.kubernetes.io/ssl-redirect: '443' tls: enabled: true # Resource allocation optimized for m5.xlarge instances resources: replicas: 3 requests: cpu: "1500m" memory: "3Gi" limits: cpu: "3000m" memory: "6Gi" # Auto-scaling configuration autoscaling: enabled: true minReplicas: 3 maxReplicas: 10 targetCPUUtilizationPercentage: 70 targetMemoryUtilizationPercentage: 80 scaleUpStabilizationWindowSeconds: 300 scaleDownStabilizationWindowSeconds: 600 # AWS-specific security settings security: podSecurityPolicy: enabled: true networkPolicy: enabled: true rbac: enabled: true serviceAccount: create: true annotations: eks.amazonaws.com/role-arn: "arn:aws:iam::$(aws sts get-caller-identity --query Account --output text):role/rtf-service-role" # Monitoring and observability monitoring: prometheus: enabled: true storageClass: "rtf-gp3-storage" grafana: enabled: true persistence: enabled: true storageClass: "rtf-gp2-storage" logging: level: "INFO" cloudwatchLogs: enabled: true region: "$AWS_REGION" logGroup: "/eks/$CLUSTER_NAME/rtf" EOF
  3. 🎯 Install Runtime Fabric Components
    # Install RTF using the installer ./install-rtf-aws.sh \ --config rtf-aws-config.yaml \ --namespace rtf-system \ --provider aws \ --region $AWS_REGION # Alternative: Helm installation with AWS-specific values helm repo add mulesoft https://repository.mulesoft.org/nexus/content/repositories/public-helm/ helm repo update # Create Helm values file for AWS cat << EOF > rtf-aws-values.yaml global: provider: aws region: $AWS_REGION storageClass: rtf-gp3-storage ingress: enabled: true className: alb annotations: kubernetes.io/ingress.class: alb alb.ingress.kubernetes.io/scheme: internet-facing alb.ingress.kubernetes.io/target-type: ip autoscaling: enabled: true minReplicas: 3 maxReplicas: 10 monitoring: enabled: true prometheus: enabled: true grafana: enabled: true logging: cloudwatch: enabled: true region: $AWS_REGION EOF # Install RTF with Helm helm install rtf-aws mulesoft/runtime-fabric \ --namespace rtf-system \ --values rtf-aws-values.yaml \ --wait \ --timeout 30m # Monitor installation progress kubectl get pods -n rtf-system -w
  4. ✅ Verify RTF Installation
    # Check RTF pod status kubectl get pods -n rtf-system -o wide # Verify services and endpoints kubectl get services -n rtf-system kubectl get endpoints -n rtf-system # Check RTF custom resources kubectl get rtf -n rtf-system -o yaml # Verify persistent volume claims kubectl get pvc -n rtf-system # Check RTF agent connectivity kubectl logs -n rtf-system -l app=rtf-agent --tail=100 # Verify load balancer kubectl get ingress -n rtf-system # Check node resources kubectl top nodes kubectl describe nodes | grep -A 5 "Allocated resources" # Verify cluster autoscaler kubectl get deployment cluster-autoscaler -n kube-system kubectl logs -n kube-system -l app=cluster-autoscaler

🌐 AWS Load Balancer Controller Setup

🔧 Step 1: Install AWS Load Balancer Controller

# Create IAM service account for AWS Load Balancer Controller eksctl create iamserviceaccount \ --cluster=$CLUSTER_NAME \ --namespace=kube-system \ --name=aws-load-balancer-controller \ --role-name AmazonEKSLoadBalancerControllerRole \ --attach-policy-arn=arn:aws:iam::aws:policy/ElasticLoadBalancingFullAccess \ --override-existing-serviceaccounts \ --approve # Add EKS Helm repository helm repo add eks https://aws.github.io/eks-charts helm repo update # Install AWS Load Balancer Controller helm install aws-load-balancer-controller eks/aws-load-balancer-controller \ -n kube-system \ --set clusterName=$CLUSTER_NAME \ --set serviceAccount.create=false \ --set serviceAccount.name=aws-load-balancer-controller \ --set region=$AWS_REGION \ --set vpcId=$VPC_ID #
