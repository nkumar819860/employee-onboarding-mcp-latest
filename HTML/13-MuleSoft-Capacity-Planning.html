📊 Recommended Configuration

📐 Core Size Estimation Formulas

1. Base Core Requirement Formula

Base_Cores = ((TPS × Payload_Factor × Complexity_Factor) / Performance_Baseline) + Overhead_Cores

Where:
  • TPS = Transactions Per Second (Peak) = Transactions_Per_Hour / 3600
  • Payload_Factor = (Payload_Size_KB / 10) + 1
  • Complexity_Factor = 1 (Simple), 1.5 (Medium), 2 (Complex), 3 (Very Complex)
  • Performance_Baseline = 1000 TPS (Simple), 500 TPS (Medium), 200 TPS (Complex), 100 TPS (Very Complex)
  • Overhead_Cores = 0.2-0.5 cores for system overhead

2. Memory Estimation Formula

Memory_GB = (Base_Cores × Memory_Per_Core) + (Concurrent_Users / 100) + (Payload_Size_MB × Buffer_Factor)

Where:
  • Memory_Per_Core = 2-4 GB (recommended), 1 GB (minimum)
  • Buffer_Factor = 10-20 depending on processing requirements

3. High Availability Scaling Formula

HA_Cores = Base_Cores × HA_Factor

Where:
  • HA_Factor = 2.0 (Active-Passive), 2.5 (Active-Active), 3.0 (Multi-Region)

🏗️ Runtime Fabric (RTF) Sizing

Total Core Sizing Formula:

RTF_Total_Cores = Application_Cores + RTF_Overhead + Kubernetes_Overhead + HA_Buffer

Where:
  • Application_Cores = Sum of all application requirements
  • RTF_Overhead = 2-4 cores for RTF services
  • Kubernetes_Overhead = 1-2 cores for K8s system pods
  • HA_Buffer = 25-50% additional for high availability and scaling

Node Sizing Recommendations:

RTF Resource Allocation Formula:

App_CPU_Limit = (Application_Cores / Total_Available_Cores) × Node_CPU_Capacity
App_Memory_Limit = (Application_Memory / Total_Available_Memory) × Node_Memory_Capacity

Cost Calculation (Annual):

Total_Annual_Cost = Infrastructure_Cost + RTF_License_Cost + Support_Cost + Operations_Cost

Where:
  • RTF_License_Cost = Production_Cores × $2,000-$3,000/core/year
  • Infrastructure_Cost = Cloud provider or hardware costs
  • Support_Cost = 22% of license cost annually
  • Operations_Cost = Personnel + Maintenance (20-30% of total)

🔄 Hybrid Deployment Sizing

Hybrid Core Distribution Formula:

Cloud_Cores = Total_Cores × Cloud_Ratio
On_Premises_Cores = Total_Cores × (1 - Cloud_Ratio)

Common Distribution Strategies:
  • Cloud-First (80/20): Public APIs, batch processing → Cloud
  • Balanced (60/40): Mixed workload distribution
  • On-Premises Heavy (40/60): Security-sensitive, legacy systems → On-premises

🏢 Private Cloud Edition (PCE) Sizing

PCE Infrastructure Sizing Formula:

PCE_Total_Cores = Runtime_Cores + Platform_Cores + Database_Cores + OS_Overhead + HA_Buffer

Where:
  • Runtime_Cores = Application runtime requirements
  • Platform_Cores = 8-16 cores for Anypoint Platform components
  • Database_Cores = 4-8 cores for database cluster
  • OS_Overhead = 10-15% of total cores
  • HA_Buffer = 50-100% for high availability and disaster recovery

PCE Total Cost of Ownership (TCO) Formula:

Annual_TCO = License_Cost + Infrastructure_Cost + Operations_Cost + Support_Cost + DR_Cost

Detailed Breakdown:
  • License_Cost = Platform license + Runtime licenses (Contact MuleSoft)
  • Infrastructure_Cost = Hardware + Network + Data Center ($200-$500/core/year)
  • Operations_Cost = Personnel + Maintenance (30-40% of infrastructure)
  • Support_Cost = 22% of license cost annually
  • DR_Cost = Disaster recovery site (50-100% of primary site)

Break-even Analysis Formula:

Break_Even_Point = PCE_Setup_Cost / (CloudHub_Annual_Cost - PCE_Annual_Operating_Cost)
Typically occurs at 100-200+ vCores depending on usage patterns

⚡ Performance Guidelines & Scaling Factors

Environment Scaling Factors

  • Development: 0.3-0.5x production sizing
  • Testing: 0.5-0.7x production sizing
  • Staging: 0.8-1.0x production sizing
  • Production: Base + 25-50% buffer
  • Disaster Recovery: 50-100% of production

Performance Baselines (per vCore)

  • Simple API: 800-1200 TPS
  • Medium Transformation: 300-500 TPS
  • Complex Processing: 100-200 TPS
  • Heavy Compute: 25-50 TPS
  • Database Operations: 200-400 TPS

Memory Guidelines

  • Minimum: 1 GB per vCore
  • Recommended: 2-4 GB per vCore
  • Heavy Payload: 4-8 GB per vCore
  • Batch Processing: 8-16 GB per vCore
  • In-Memory Processing: 16+ GB per vCore

High Availability Multipliers

  • Single Zone: 1.5x (buffer only)
  • Multi-AZ: 2.0x minimum
  • Multi-Region: 2.5-3.0x
  • Active-Active: 2.0x per site
  • Global DR: 3.0-4.0x total capacity

💰 Cost Optimization Strategies

CloudHub Optimization

  • Right-sizing: Monitor CPU/memory usage
  • Reserved Instances: 20-30% discount
  • Environment Consolidation: Combine dev/test
  • Auto-scaling: Scale down during off-hours
  • Worker Sharing: Multiple apps per worker

RTF Optimization

  • Node Efficiency: Maximize utilization >80%
  • Spot Instances: 50-70% cost reduction
  • Resource Limits: Prevent resource waste
  • Multi-tenancy: Share infrastructure
  • Auto-scaling: HPA and VPA policies

Hybrid Strategies

  • Workload Placement: Cost-optimal locations
  • Cloud Bursting: On-demand scaling
  • Data Locality: Reduce transfer costs
  • License Optimization: Shared licensing
  • Network Efficiency: Optimize connectivity

PCE Efficiency

  • Infrastructure Sharing: Multi-tenant design
  • Lifecycle Management: Hardware refresh cycles
  • Energy Efficiency: Green data centers
  • Automation: Reduce operational overhead
  • Capacity Planning: Avoid over-provisioning
